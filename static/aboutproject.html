About the Project

This project showcases my complete **DevOps pipeline automation**, built with modern cloud technologies to demonstrate continuous integration, continuous deployment, and infrastructure management.

Project Overview

The objective was to automate the deployment of a containerized web application on AWS, using **Docker**, **Kubernetes (EKS)**, and **ArgoCD**, ensuring zero-downtime rollouts and robust monitoring. The CI/CD pipeline was designed to handle end-to-end delivery — from code commits to live production updates.

Tools & Technologies Used

  • GitHub Actions – for CI/CD automation and build pipelines
  • Docker – for containerizing the application
  • Helm – for packaging and deploying Kubernetes manifests
  • ArgoCD – for GitOps-based continuous deployment
  • AWS EKS – for scalable Kubernetes cluster hosting
  • Terraform – for infrastructure provisioning (IaC)

Workflow Summary

  1. Code changes are committed and pushed to the GitHub repository.
  2. GitHub Actions automatically builds the latest Docker image and pushes it to Docker Hub.
  3. Helm chart values are updated with the latest image tag.
  4. Argo CD continuously monitors the Helm chart repository for changes.
  5. When a new image version is detected, Argo CD automatically deploys the update to the Kubernetes cluster in EKS.
  6. The app becomes available via an AWS LoadBalancer with an ingress controller managing routing.

Features Implemented

  • Automated Docker image build and push using GitHub Actions.
  • Dynamic image version updates in Helm charts.
  • Argo CD setup for GitOps-based deployment automation.
  • Ingress setup for external application access through AWS ELB.
  • Minikube used for local testing before production deployment.
  • Declarative and reproducible infrastructure using YAML manifests.

Key Learnings

Through this project, I learned to connect multiple DevOps tools into one automated ecosystem. The project strengthened my understanding of:

  • How CI/CD pipelines are designed and automated end-to-end.
  • How Argo CD enables GitOps-style continuous deployment.
  • How Helm simplifies versioned Kubernetes deployments.
  • Best practices for securing and managing workloads in AWS EKS.

Outcome

The final system enables rapid, reliable, and repeatable deployments across environments. It minimizes manual intervention, improves team velocity, and ensures high availability of applications.
